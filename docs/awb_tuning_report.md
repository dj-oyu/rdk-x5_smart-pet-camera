# 夜間カメラ AWB チューニングレポート

- 日付: 2026-03-21
- 対象: Camera 1 (NIGHT/IR, IMX219, IRカットフィルタなし)
- ツール: `build/test_awb_tuning --camera 1`
- テストスクリプト: `scripts/test_awb_patterns.sh`
- 保存フレーム: `test_pic/awb_260321_*.jpg` (NV12も同名で保存)

## 背景・課題

- 夜間IRカメラの映像が紫色になる（AWB Autoのデフォルトゲイン R=1.91, B=2.13）
- 少し明かりをつけると青くなり、暗くなっても青いまま戻らない
- 原因: IR映像には意味のある色情報がなく、AWB Autoが収束できずゲインがドリフトする

## ISP API 対応状況 (RDK X5 + IMX219)

| API | 関数 | 結果 |
|---|---|---|
| AWB (ホワイトバランス) | `hbn_isp_set_awb_attr` | **有効** - Manual/Auto切替、ゲイン調整が映像に反映される |
| WDR (ワイドダイナミックレンジ) | `hbn_isp_set_wdr_attr` | **無効** - error -65545 |
| Gamma (ガンマ補正) | `hbn_isp_set_gc_attr` | **無効** - error -65545 (gamma=1.0のみ成功するが映像変化なし) |
| Color Process (輝度/コントラスト) | `hbn_isp_set_color_process_attr` | **無効** - API成功するがget値が変わらず映像に反映されない |
| 3DNR (時間ノイズリダクション) | `hbn_isp_set_3dnr_attr` | **有効** - 既存実装で確認済み |

### AWBゲインの制約

- ゲイン値は **1.0以上** が必要。1.0未満を指定すると error -65545
- `rgain=0.8` → 失敗、`rgain=1.0` → 成功
- 最大値は少なくとも3.25まで確認（それ以上は未検証）

## 採用設定

```
AWB Mode: MANUAL
rgain:  1.8
grgain: 1.8
gbgain: 1.8
bgain:  2.34
```

- 色味: 02_cool_lightと同じR:G:B = 1:1:1.3比率
- 暗部: 全体ゲイン1.8倍で暗部が十分見える
- ノイズ: 3DNR=128（既存設定）との組み合わせで許容範囲

## 再テスト方法

```bash
# 1. カメラデーモンを停止
pkill -f camera_switcher_daemon; sleep 1; pkill -f camera_daemon_drobotics

# 2. バッチテスト実行
bash scripts/test_awb_patterns.sh

# 3. インタラクティブモードで試行錯誤
./build/test_awb_tuning --camera 1
# awb> m 1.8 1.8 2.34   ← 設定
# awb> s my_test         ← フレーム保存
# awb> d                 ← 現在値ダンプ

# 4. フレーム確認
ffplay -f rawvideo -pix_fmt nv12 -s 1920x1080 test_pic/awb_*.nv12
# または JPG で確認（ffmpegで変換済み）

# 5. カメラデーモン再起動
cd src/capture && make switcher-daemon
```

## 今後の調整ポイント

- **比率 (B/R)**: 現在1.3。青みを強くしたければ1.4〜1.5に。ニュートラルにしたければ1.0に
- **全体ゲイン**: 現在1.8。明るくしたければ2.0に上げる（ノイズとトレードオフ）
- **暗部持ち上げ**: ISP側では不可（WDR/Gamma/CPROC全て無効）。YOLO側のCLAHEで対応
- **AWB Auto偏好**: `auto_attr.rg_strength` / `bg_strength` でAutoのバイアスを変えられる可能性あり（未検証）
