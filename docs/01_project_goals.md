# プロジェクトゴール - スマートペットカメラ

## プロジェクト概要

AI物体検出技術を活用し、ペット（猫）の日常行動を自動的に記録・分析するスマート監視システム。
D-Robotics RDK X5上で稼働中。昼夜2カメラ体制でリアルタイム検出・WebRTCストリーミング・アルバム自動生成まで実装済み。

## 背景と目的

### 背景
- ペットの健康管理には日々の食事量や水分摂取量の把握が重要
- 飼い主が不在時のペットの行動を記録したいニーズがある
- 手動での記録は手間がかかり、継続が難しい

### 目的
- 猫の水飲み・食事行動を自動的に検出・記録
- 24時間体制での監視（昼夜自動切替）
- 行動データの可視化と分析基盤の構築

## 現在の稼働状態

### [実装済] コア機能
1. **リアルタイム物体検出システム** - YOLOv11n（3.3MB）をBPU（10 TOPS INT8）で推論、8.9msレイテンシ
2. **昼夜2カメラ自動切替** - 明るさベースで昼間カメラ/夜間IRカメラを自動選択
3. **WebRTC/MJPEGストリーミング** - Preact Web UIからリアルタイム視聴（WebRTC遅延 <100ms）
4. **アルバム自動キャプチャ** - 検出イベント時に4コマパネル画像を自動生成・保存
5. **行動推定** - バウンディングボックス重なり判定による食事・水飲み行動の推定

### [実装済] 追加機能
1. **VLM連携** - M5Stack AI Pyramid Pro (AX8850) 上のRust実装 (axum + rusqlite)。Qwen3-VL-2Bによる行動解析・キャプション生成
2. **pet_id個体識別** - HSVカラー分析によるリアルタイム判定（Go bbox色分類 → ai-pyramid連携）
3. **H.265録画パイプライン** - NALキャプチャ → .hevc → ffmpeg → .mp4、サムネイル自動生成
4. **ヒートマップSSE** - 10fpsリアルタイムプッシュ配信

### [未実装] 計画中の機能
1. **JSONイベント記録** - スキーマ定義済みだがコード未実装
2. **行動パターン可視化ダッシュボード**
3. **統一YAML設定ファイル** - 現状パラメータはハードコード

### ビジネス価値
- ペットの健康状態の早期発見
- 飼い主の安心感向上
- 獣医診断時の客観的データ提供

## プロジェクトスコープ

### 対象範囲
- 猫の食事・水飲み行動の検出と記録
- 昼間カメラ・夜間IRカメラの2基体制での24時間監視
- YOLOv11nによる物体検出とバウンディングボックス重なり判定による行動推定
- WebRTC/MJPEGによるリアルタイムストリーミング
- アルバム（4コマパネル）の自動生成

### 対象外
- リアルタイムアラート機能（将来的に検討）
- クラウド連携機能（将来的に検討）
- 複数のペット個体識別（将来的に検討）
- スマートフォンネイティブアプリ開発（Web UIで代替）

## 技術的な前提

### ハードウェア
- **ボード**: D-Robotics RDK X5（ARM, 8GB RAM）
- **BPU**: 10 TOPS INT8（シングルコア）
- **GPU**: Vivante GC8000L（6.75 GFLOPS、OpenCL zero-copy用途のみ実用的）
- **カメラ**: 昼間用カメラ + 夜間IR対応カメラの2基体制
- **映像キャプチャ**: hbn_vflow API (VIN→ISP→VSE) + hb_mm_mc H.265エンコーダによる共有メモリ連携

### ソフトウェア
- C言語によるカメラデーモン（`src/capture/`）
- Pythonによる物体検出 (hobot_dnn BPU推論)
- Goによるストリーミングサーバー (pion/webrtc v4)
- YOLOv11n（BPU推論、`hobot_dnn`ランタイム）
- Preact Web UI（WebRTC/MJPEGビューア + アルバム閲覧）
- パッケージ管理: `uv`

## 成功基準

1. 食事・水飲み行動の検出精度：90%以上
2. 24時間連続稼働の安定性 [実装済]
3. 記録データの正確性と完全性
4. 昼間カメラ: 30fps維持 [実装済]
5. YOLO推論: 33ms以内（実測8.9ms） [実装済]
6. WebRTC遅延: 100ms以内 [実装済]
