# 要件定義 - スマートペットカメラ

## 機能要件

### FR-01: 映像キャプチャ機能 [実装済]
- **FR-01-01**: システムは2基のカメラ（昼間用・夜間IR用）から映像をキャプチャできる [実装済]
- **FR-01-02**: 明るさベースで昼間/夜間カメラを自動切替する [実装済]
- **FR-01-03**: 昼間カメラは30fpsのフレームレートを維持する [実装済]
- **FR-01-04**: キャプチャした映像は共有メモリ経由でリアルタイムに処理パイプラインに渡される [実装済]
- [制約事項] 夜間カメラの実効検出フレームレートは約22fps（ROI 3フレームサイクル方式）

### FR-02: 物体検出機能 [実装済]
- **FR-02-01**: システムは映像フレームから猫の体を検出できる [実装済]
- **FR-02-02**: システムは餌皿（食事エリア）を検出できる [実装済]
- **FR-02-03**: システムは水飲み場を検出できる [実装済]
- **FR-02-04**: 検出結果にはバウンディングボックス（座標、幅、高さ）とクラス名、信頼度が含まれる [実装済]
- **FR-02-05**: 物体検出の推論時間はフレーム間隔内（33ms以内）に完了する [実装済] - 実測: BPU 8.9ms + Python API <1ms

#### 採用モデル
- **YOLOv11n**: 3.3MB、BPU推論8.9ms（10 TOPS INT8シングルコア）
- 評価済み代替: YOLOv8n, YOLOv13n, YOLOv26n（詳細は `detection-and-yolo.md` 参照）

### FR-03: 行動推定機能 [実装済]
- **FR-03-01**: 猫のバウンディングボックスと餌皿のバウンディングボックスの重なり具合から食事行動を推定する [実装済]
- **FR-03-02**: 猫のバウンディングボックスと水飲み場のバウンディングボックスの重なり具合から水飲み行動を推定する [実装済]
- **FR-03-03**: 重なり閾値（IoU: Intersection over Union等）を設定可能とする [実装済]
- **FR-03-04**: 行動開始時刻と終了時刻を記録する [実装済]
- **FR-03-05**: 誤検出を減らすため、一定時間継続した行動のみを記録する [実装済]

### FR-04: データ記録機能 [一部未実装]
- **FR-04-01**: 検出された行動イベントをJSON形式で記録する [未実装] - スキーマ定義済みだがコード未実装
- **FR-04-02**: JSON記録には以下の情報を含める： [未実装]
  - タイムスタンプ
  - 行動種別（食事/水飲み）
  - 行動開始時刻
  - 行動終了時刻
  - 信頼度
  - 使用カメラID
- **FR-04-03**: 行動イベント発生時の映像を動画ファイルとして保存する [未実装]
- **FR-04-04**: 動画ファイルは行動開始数秒前から終了数秒後まで含める [未実装]
- **FR-04-05**: 動画ファイル名とJSONデータは関連付け可能な命名規則を持つ [未実装]

### FR-04B: アルバム自動キャプチャ機能 [実装済]
- **FR-04B-01**: 検出イベント時に4コマパネル画像を自動生成する [実装済]
- **FR-04B-02**: Web UIからアルバムを閲覧できる [実装済]

### FR-05: データ管理機能 [未実装]
- **FR-05-01**: 記録データは日付ごとにディレクトリ分けして保存する
- **FR-05-02**: ストレージ容量を監視し、空き容量が一定以下になったら警告を出す
- **FR-05-03**: 古いデータを自動削除する機能を提供する（設定可能な保持期間）

### FR-06: 設定管理機能 [未実装]
- **FR-06-01**: カメラパラメータ（解像度、フレームレート等）を設定ファイルで管理できる [未実装] - 現状ハードコード
- **FR-06-02**: 物体検出モデルのパスと設定を指定できる [未実装] - 現状ハードコード
- **FR-06-03**: 行動推定の閾値を設定できる [未実装] - 現状ハードコード
- **FR-06-04**: 記録保存先のパスを設定できる [未実装] - 現状ハードコード
- [未実装] 統一YAML設定ファイルへの移行が必要

### FR-07: ストリーミング機能 [実装済]
- **FR-07-01**: WebRTCによるリアルタイム映像ストリーミング [実装済] - 遅延 <100ms
- **FR-07-02**: MJPEGフォールバックストリーミング [実装済]
- **FR-07-03**: Preact Web UIによるビューア [実装済]
- [制約事項] H.265ビットレート上限: 700kbps（hb_mm_mc制約）

## 非機能要件

### NFR-01: パフォーマンス [実装済]
- **NFR-01-01**: システムは24時間連続稼働可能である [実装済]
- **NFR-01-02**: メモリリークが発生しない [実装済]
- **NFR-01-03**: CPU使用率（アイドル時）: 実測約0.7%（目標80%以下を大幅に達成） [実装済]
- **NFR-01-04**: CPU使用率（WebRTCストリーミング時）: 110-130% [制約事項] - マルチコア合算値
- **NFR-01-05**: YOLO推論遅延: 実測8.9ms（BPU） + <1ms（Python API） [実装済]
- **NFR-01-06**: WebRTC遅延: <100ms [実装済]
- **NFR-01-07**: 昼間カメラ: 30fps維持 [実装済]
- **NFR-01-08**: 夜間カメラ実効検出: 約22fps（ROI 3フレームサイクル） [制約事項]

### NFR-02: 信頼性 [一部未実装]

- **NFR-02-02**: データ記録中の電源断に対して、可能な限りデータを保護する
- **NFR-02-03**: カメラ接続エラー時にリトライ処理を行う [実装済]
- **NFR-02-04**: エラーログを記録し、デバッグを容易にする [実装済]

### NFR-03: 保守性
- **NFR-03-01**: コードはモジュール化され、各機能が独立している [実装済] - capture/detector/streaming_server/common分離

- **NFR-03-03**: ログ出力は適切なレベルで分類される [実装済]
- **NFR-03-04**: ドキュメントが整備され、システムの理解が容易である [実装済]

### NFR-04: 拡張性

- **NFR-04-03**: マルチカメラ融合モード [未実装]

## システム要件

### ハードウェア要件
- **HW-01**: カメラモジュール x 2（昼間用 + 夜間IR対応） [実装済]
- **HW-02**: D-Robotics RDK X5（ARM Cortex-A55）
- **HW-03**: メモリ: 8GB RAM
- **HW-04**: ストレージ: microSD（推奨64GB以上）
- **HW-05**: BPU: 10 TOPS INT8（シングルコア） [制約事項]
- **HW-06**: GPU: Vivante GC8000L - 6.75 GFLOPS [制約事項] OpenCL zero-copy用途のみ実用的

### ハードウェア制約事項
- [制約事項] H.265エンコードビットレート上限: 700kbps（hb_mm_mc制約）
- [制約事項] BPUはシングルコアのみ使用可能
- [制約事項] ISPランタイムAPI制限: AWB/3DNR/2DNRのみ動作。Gamma/WDR/CPROCはランタイム変更不可
- [制約事項] AWBはISP起動後30フレーム経過後に適用が必要
- [制約事項] 夜間カメラのAWBはMANUALモード必須（Auto AWBはIRカメラで動作しない）

### ソフトウェア要件
- **SW-01**: OS: Linux（RDK X5標準イメージ）
- **SW-02**: カメラパイプライン: hbn_vflow API (VIN→ISP→VSE→Encoder)
- **SW-03**: C/C++コンパイラ: GCC（ARM向け）
- **SW-04**: Python: 3.11+（`uv`によるパッケージ管理）
- **SW-05**: OpenCV: 4.x以上
- **SW-06**: 推論ランタイム: `hobot_dnn`（BPU専用）

### ネットワーク要件
- **NW-01**: WebRTC/MJPEGストリーミングのためLAN接続が必要 [実装済]
- **NW-02**: 外部ネットワーク接続は不要（オフライン動作可能）

## 制約事項

### C-01: ハードウェア制約
- BPUはシングルコア（10 TOPS INT8）のみ。マルチモデル同時推論は不可
- H.265ビットレート上限700kbps（hb_mm_mc制約）
- ISPランタイムAPI制限（AWB/3DNR/2DNRのみ、Gamma/WDR/CPROC不可）
- AWBはISP起動30フレーム後に適用必須
- 夜間IR カメラはAWB MANUALモード必須
- GPU（Vivante GC8000L）は6.75 GFLOPSで実用性限定的

### C-02: 環境的制約
- 照明条件が変化する（昼夜、室内照明のON/OFF） - 明るさベース自動切替で対応済み
- カメラ位置は固定される

### C-03: 運用的制約
- プロセス自動復旧・クリーンアップが未実装
- 設定変更にはコード修正が必要（統一YAML設定未実装）
- ネットワーク接続がない環境でもローカル動作する
